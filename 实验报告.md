# 学生成绩管理系统实验报告

## 1. 实验目的

- 掌握基于Vue 3和Express的全栈Web应用开发
- 学习MySQL数据库的设计和使用
- 理解前后端分离架构的实现
- 掌握API接口的设计和测试
- 学习项目结构的优化和文档编写

## 2. 实验环境

- **操作系统**: Windows 10
- **前端技术**: Vue 3、Vue Router、Axios
- **后端技术**: Express、MySQL
- **构建工具**: Vue CLI
- **数据库**: MySQL 5.7+
- **开发工具**: VS Code

## 3. 实验内容

### 3.1 项目结构设计

```

│   ├── models/            # 数据库模型
│   │   ├── db.js          # 数据库连接和初始化
│   │   └── Student.js     # 学生模型（业务逻辑）
│   ├── routes/            # API路由
│   │   └── studentRoutes.js  # 学生相关API路由
│   ├── index.js           # 主入口文件
│   ├── package-lock.json  # 依赖锁文件
│   └── package.json       # 后端依赖
├── vuestu/                # 前端代码
│   ├── public/            # 静态资源
│   │   ├── favicon.ico    # 网站图标
│   │   └── index.html     # 前端入口HTML
│   ├── src/               # 源代码

│   │   │   └── index.js    # 路由配置
│   │   ├── views/          # 视图
│   │   │   └── HomeView.vue  # 主页
│   │   ├── App.vue         # 根组件
```

### 3.2 数据库设计

#### 3.2.1 数据库表结构

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `INT` | `PRIMARY KEY` | 学号，5位数字 |
| `name` | `VARCHAR(50)` | `NOT NULL` | 姓名 |
| `gender` | `BIT(1)` | `NOT NULL DEFAULT 0` | 性别：0-女，1-男 |
| `chinese` | `INT` | `NOT NULL DEFAULT 0` | 语文成绩，范围0-100 |
| `math` | `INT` | `NOT NULL DEFAULT 0` | 数学成绩，范围0-100 |
| `english` | `INT` | `NOT NULL DEFAULT 0` | 英语成绩，范围0-100 |
| `total` | `INT` | `DEFAULT 0` | 总分 |
| `average` | `DECIMAL(5,2)` | `DEFAULT 0.00` | 平均分 |

#### 3.2.2 数据库初始化

系统支持自动数据库初始化，当后端服务器启动时，会自动：
- 创建数据库（如果不存在）
- 创建表结构（如果不存在）
- 添加必要的字段（如果不存在）
- 插入测试数据（如果表为空）
- 计算并更新总分和平均分

### 3.3 后端API设计

| API路径 | 方法 | 功能 | 请求体 | 响应 |
| :--- | :--- | :--- | :--- | :--- |
| `/list_user` | `GET` | 获取所有学生成绩 | 无 | 学生成绩列表（按平均分降序） |
| `/insert` | `POST` | 添加学生成绩 | `{id, name, gender, chinese, math, english}` | `{success: true, message: "添加成功", id: 插入的ID}` |
| `/edit` | `POST` | 编辑学生成绩 | `{id, name, gender, chinese, math, english}` | `{success: true, message: "修改成功", affectedRows: 影响的行数}` |
| `/delete/:id` | `DELETE` | 删除学生成绩 | 无 | `{success: true, message: "删除成功", affectedRows: 影响的行数}` |
| `/get_user` | `GET` | 获取单个学生成绩 | 无（参数通过查询字符串传递） | 学生信息 |

### 3.4 前端功能设计

#### 3.4.1 组件设计

- **StudentInfo.vue**: 学生列表组件，显示所有学生成绩，支持按平均分排序，提供添加、编辑、删除功能
- **StudentInsert.vue**: 添加学生组件，支持学号和成绩验证
- **StudentEdit.vue**: 编辑学生组件，支持成绩验证

#### 3.4.2 路由设计

| 路径 | 组件 | 功能 |
| :--- | :--- | :--- |
| `/` | `HomeView` | 主页 |
| `/info` | `StudentInfo` | 学生列表 |
| `/insert` | `StudentInsert` | 添加学生 |
| `/edit/:id` | `StudentEdit` | 编辑学生 |

#### 3.4.3 验证功能

- 学号限定为5位数字
- 成绩范围为[0，100]
- 性别转为男女选择

### 3.5 项目优化

#### 3.5.1 项目结构优化

- 采用模块化设计，将后端代码按照功能划分为配置、模型、路由等模块
- 前端代码按照组件化的思想进行组织
- 删除无用的文件和文件夹，减少项目体积

#### 3.5.2 性能优化

- 后端采用异步处理，提高并发能力
- 前端使用Vue 3的Composition API，提高代码可维护性
- 数据库查询使用索引，提高查询效率

#### 3.5.3 用户体验优化

- 美观的用户界面
- 响应式设计，支持不同屏幕尺寸
- 流畅的交互体验
- 清晰的错误提示

## 4. 实验步骤

### 4.1 环境搭建

1. 安装Node.js 14+
2. 安装MySQL 5.7+
3. 安装Vue CLI

### 4.2 项目初始化

1. 创建后端项目：
   ```bash
   mkdir backend
   cd backend
   npm init -y
   npm install express mysql body-parser cors
   ```

2. 创建前端项目：
   ```bash
   vue create vuestu
   cd vuestu
   npm install axios vue-router
   ```

### 4.3 数据库设置

1. 启动MySQL服务
2. 系统会自动创建数据库和表结构

### 4.4 项目运行

#### 4.4.1 一键启动（推荐）

1. 安装依赖：
   ```bash
   # 安装后端依赖
   cd backend && npm install
   # 安装前端依赖
   cd ../vuestu && npm install
   ```

2. 运行一键启动脚本：
   ```bash
   # 在项目根目录运行
   start.bat
   ```

#### 4.4.2 手动启动

1. 启动后端：
   ```bash
   cd backend
   node index.js
   ```

2. 启动前端：
   ```bash
   cd vuestu
   npm run serve
   ```

### 4.5 功能测试

1. 运行API测试脚本：
   ```bash
   # 安装测试脚本依赖
   npm install axios
   # 运行测试脚本
   node test_api.js
   ```

2. 手动测试：
   - 访问前端页面：http://localhost:8080/
   - 测试添加学生功能
   - 测试编辑学生功能
   - 测试删除学生功能
   - 测试排序功能
